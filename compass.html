The Touchstone Master Evaluation Framework

Three gates, always in this order. Everything clears Gate 1 before Gate 2 before Gate 3.

Gate 1 — Is it safe? Safety & Security

Binary pass/fail. Clears first. Two sides: Safety (authorized agents acting wrong) and Security (unauthorized access). Nothing else matters until this gate passes.

As a capacity: Judgment — the capacity to ask "should this exist?" before "how should this work?"
Gate 2 — Does it work? Effectiveness

Threshold — must work before you optimize. The thing has to actually do what it's supposed to do. No point making something fast if it doesn't work yet.

As a capacity: Precision — encoding intent clearly enough that downstream actors can execute without reconstructing your reasoning.
Gate 3 — Is it efficient? Efficiency

Continuous improvement — the reward for clearing Gates 1 and 2. Now make it fast, cheap, and elegant. This is where optimization lives.

As a capacity: Execution — deliver efficiently against a clear, well-judged spec.

Map Operations Structural Transformation Vocabulary

The shared language for how you and Claude change any plan, roadmap, or knowledge structure. Every operation is a clarification of shared understanding.

Operation Pair Type What It Does
Promote / Demote Vertical movement Move something up or down in priority or scope level
Merge / Split Structural Combine two things into one, or break one thing apart. Original identity is destroyed.
Rename / Alias Identity Replace a name (Rename) or add an additional name (Alias) without replacing the original
Group / Ungroup Containment Add or remove a boundary around items. Items inside are preserved. Can be ordered or unordered.
Add / Remove Existence Create a new thing or delete an existing thing
Reorder Sequence (independent) Change the order. Logics: Priority, Sequential, Dependency, Scope, Specificity, Arbitrary.
